## 🎯 Agent 重点

### 1. Anthropic 拟以 9000 亿美元估值新一轮融资，估值首次反超 OpenAI
- **要点**：Claude 母公司 Anthropic 正洽谈新一轮融资，估值或超 9000 亿美元，并最快 10 月推进 IPO。
- **详情**：据 The Information、Bloomberg 等多家媒体报道，Anthropic 与多家主权基金、大型机构投资者就新一轮融资进行接触，目标估值超过 **9000 亿美元**，将首次在私募市场上反超 OpenAI（最新私募估值约 8520 亿美元）。Anthropic 同时被报道考虑最快于 **2026 年 10 月**进行首次公开募股，本轮融资被视为 IPO 前的"压轴老股+新股"组合。背后逻辑是 Claude 在企业代码与 Agent 工作流的渗透——其 ARR 已突破 **百亿美元**门槛。
- **影响**：AI 大模型的二级竞争格局正在从"模型能力差距"过渡到"企业工作流深度嵌入"。Anthropic 估值反超意味着市场更看重 Agent / Coding 商业化变现而非通用聊天，这将进一步推高 Claude Code、MCP 生态相关上下游公司的估值与并购溢价。
- 🔗 [The Information / 界面新闻 - Anthropic 9000 亿美元估值融资](https://www.jiemian.com/article/13076456.html)

### 2. Google I/O 2026 全面押注 Agent：Gemini 3.5 Flash + Antigravity 2.0 + UCP/AP2 协议
- **要点**：Google 把 Gemini App、搜索 AI 模式的默认模型切到 Gemini 3.5 Flash，同时发布 Agent 开发平台 Antigravity 2.0 与跨 Agent 通信协议 UCP/AP2。
- **详情**：在 5 月 19-20 日的 Google I/O 2026 上，Google 一次性发布三件套：（1）**Gemini 3.5 Flash** 主打速度与低成本，成为 Gemini App / Search AI Mode 的默认模型，3.5 Pro 将于 6 月上线；（2）多模态创作模型 **Gemini Omni** 支持文/图/音/视频混合编辑；（3）**Antigravity 2.0** 作为"Agent 操作系统级"开发平台，配套 UCP（Universal Context Protocol）和 AP2（Agent Payments Protocol v2）两套协议，对标 Anthropic MCP + Google 自家 A2A 草案。Google 同时演示了消费者直接用 Agent 完成"订餐 / 订票 / 比价 / 售后"的端到端任务。
- **影响**：Google 把战略重心从"聊天机器人"转向"主动执行任务的 AI Agent"，与 Anthropic Computer Use、OpenAI Operator 形成正面竞争。UCP/AP2 协议如果开放，可能成为继 MCP 之后第二个事实标准，Agent 间的支付、身份与上下文传递将首次有了底层规范。
- 🔗 [TechCrunch - Google bets next AI wave on agents](https://techcrunch.com/2026/05/19/with-gemini-3-5-flash-google-bets-its-next-ai-wave-on-agents-not-chatbots/)
- 🔗 [TechCrunch - Google 向消费者推销 Agent 生态](https://techcrunch.com/2026/05/21/google-is-pitching-an-ai-agent-ecosystem-to-consumers-who-may-not-buy-it/)

### 3. Anthropic 收购 SDK / MCP 平台 Stainless，竞品 SDK 生成基础设施被切断
- **要点**：Anthropic 收购为 OpenAI、Stripe 等多家公司提供 SDK 自动生成的 Stainless，并将关停其面向外部的托管服务。
- **详情**：Stainless 是一家从 OpenAPI Spec 自动生成多语言 SDK 与 MCP Server 的基础设施公司，由 Stripe 前工程师 Alex Rattray 创立，长期为 **OpenAI、Anthropic、Stripe、Cloudflare** 等头部公司生成官方 SDK。Anthropic 收购后宣布将"逐步关停 Stainless 所有面向外部的托管服务"，仅保留 Anthropic 内部使用——这意味着 OpenAI 等竞争对手将无法继续使用该自动化基础设施，需自行重建或迁移。Stainless 目前服务超 **3000 个 MCP server** 的代码生成。
- **影响**：这不是一次普通的人才并购，而是一次"基础设施级断供"。Anthropic 把开发者的 `import` 语句锁定在自己的生态里——谁拥有 SDK，谁就拥有开发者的依赖路径。MCP 生态的话语权进一步向 Anthropic 倾斜。
- 🔗 [Anthropic 官方公告 - Anthropic acquires Stainless](https://www.anthropic.com/news/anthropic-acquires-stainless)

### 4. xAI 上线 Grok Skills：单次问答转向"可配置自动化工作空间"
- **要点**：xAI 在网页端、iOS 与 Android 同步上线 Grok Skills，赋予 Grok 跨对话的持久记忆与可配置技能。
- **详情**：5 月 19 日，xAI 宣布上线 **Skills** 功能，标志着 Grok 从单次问答聊天机器人正式向"可配置、可编程的自动化工作空间"演进。用户可创建命名 Skill（含系统 prompt、工具列表、记忆配置），跨对话持久生效；功能对标 Anthropic Skills、OpenAI Custom GPTs。同期 xAI 发布的 **Grok Build** 是其首款 CLI 编程 Agent，支持"计划模式"——执行前可审阅 / 评论 / 重写任务方案。值得注意的是，xAI 此前被报道将整体并入 SpaceX 成为 SpaceXAI 子部门，但 Skills/Build 的发布节奏未受影响。
- **影响**：Grok 跟进 Anthropic / OpenAI 的"Skill+Agent"叙事，意味着 L3 级 Agent（带持久记忆+技能可配置）已经成为头部模型厂商的标配能力，而非差异化卖点。后续竞争将转向 Skill 生态的丰富度与互操作性。
- 🔗 [界面新闻 - xAI 推出 Skills 功能](https://www.jiemian.com/article/13072891.html)
